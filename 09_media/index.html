PNG optimization

  • pnglitch helps to understand how PNG works
  • optipng - discards garbage and applies multiple brute force compression attempts with different parameters and with the best Zlib codec
  • pngquant applies invisible lossy image changes to enable dramatically better compression; generates the best palette.
  • npm install optipng pngquant
  • Expected impact: 20-70%

Animated PNG

  • Modern alternative to GIF
  • PNG format extension to allow GIF-like animation
  • Fallback to first frame on not supporting engines
  • Supported by all browsers except Chromium-derived and IE/Edge

webp

  • Advanced image compression format optimized for web
  • Official site
  • Appeared in 2010, developed by Google
  • Similar to JPEG, but supports more features, such as transparency, animation, etc.
  • Claims to be much better, but in practice, not significantly
  • Supported only by Chromium-derived browsers
  • So the plan is to use APNG/webp for complex animations through srcset

<picture>

  • Screens of different size and DPI get the same img
  • The idea behind <picture> is to make web images responsive (Russian)
<picture>
  <source media="(min-width: 40em)"
    srcset="big.jpg 1x, big-hd.jpg 2x">
  <source
    srcset="small.jpg 1x, small-hd.jpg 2x">
  <img src="fallback.jpg" alt="">
</picture>

<video>: mp4

  • Official site
  • Part of MPEG-4
  • Proprietary, covered with patents and involves royalty payments
  • Video codec: h.265, h.264
  • Audio codec: differs, usually AAC and AC-3
  • Supported by all major browsers

<video>: webm

  • Official site
  • Appeared in 2010, originally promoted by Google
  • Open source (BSD license)
  • Video codec: VP9 ( VP8 is the previous version, VP10 is upcoming)
  • Audio codec: differs, usually Vorbis and Opus
  • Supported by all major browsers except Safari and IE/Edge (Edge: in development)

<video>: which format to choose?

  • Use <source>: 
       <video width="1920" height="1080" controls>
  <source src="movie.mp4" type="video/mp4">
  <source src="movie.webm" type="video/webm">
Your browser does not support the video tag.
</video>
  • Use ffmpeg - LGPL engine which supports both formats
