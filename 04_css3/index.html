What is CSS

  • Cascading Style Sheets
  • Specify how to visualize the DOM
  • Organized as the plain list of rules
  • Each rule is applied to certain elements via selectors
  • Each rule may change several display options
  • CSS is included into HTML page via style attribute, <style> or <link>
a {
	color: blue;
}

Selectors

  • The Standard
  • Comprise of the following types:
    • Type selector a {}
    • Universal selector * {}
    • Attribute selector [extra~=style] ... <span extra="style other">
    • Class selector .highlight {} ... <span class="highlight">
    • ID selector #special {} ... <span id="special">
  • May recursively contain other selectors via combinators
  • There are pseudo-classes for even more fine-grained control and interactivity
  • There are pseudo-elements for reverse DOM manipulation

Combinators

  • selector1,selector2 is like "or"
  • selector1selector2 is like "and"
  • selector1 selector2 is for descendants
  • selector1 > selector2 is for children (direct descendants)
  • selector1 + selector2 is for direct siblings
  • selector1 ~ selector2 is for general siblings

Pseudo-classes

  • Dynamic
    • :hover - mouse is over
    • :focus - element is focused
    • :visited - link was previously visited
  • :target
  • :lang
  • UI states
    • :enabled, :disabled
    • :checked, :indeterminate
  • Structural, e.g. :nth-child
  • :not

Pseudo-elements

  • ::before, ::after insert elements with special option "content"
  • ::first-letter, ::first-line
  • ::selection - highlighted text, only in level 4
a::after {
  content: attr(hint);
  color: red;
}
<a hint="test"></a>

CSS box model

CSS units

Link
  • Absolute: px, mm, cm, in, pt, pc
  • Font: em, ex, ch, rem
  • Viewport: vh, vw, vmin, vmax
  • %
  • auto

CSS box model options

  • margin; margin-left, margin-top, etc. May be negative.
  • border: border-style, border-width, border-color, etc.
  • padding: padding-left, padding-top, etc. May not be negative.
div#container {
  margin: 1px 2em 3% 4vh;  /* top right bottom left */
  border: 1px solid blue;
  padding: auto 5vmin;  /* top = bottom, right = left */
}

box-sizing

Sets whether padding and border must be included into width and height.

  • content-box
  • border-box

CSS layout

display

Sets which element box type is used. Link.

  • none - invisible, do not take part in the layout
  • block - no horizontal elements before and after
  • inline - like text
  • inline-block - inline box
  • flex - modern grid
  • table* - traditional grid
  • ...

flex-box

			display: flex;
  • Standard
  • Article
  • flexboxgrid - CSS framework

position

Controlls how elements are positioned. MDN, Standard

  • static - this is the default, obey the normal flow
  • relative - move element without changing the layout
  • absolute - allows setting exact coordinates (left, top, etc.) in the parent element, ignored in layout
  • fixed - allows setting exact coordinates in the viewport, element is ignored in layout
  • sticky - similar to relative, read an example, not supported by Chrome

float

  • Excludes element from the flow, MDN
  • float: left; (right)

Hiding elements

  • display: none; - changes layout
  • visibility: hidden; - saves layout, but triggers the event (opposite is visible)
  • opacity: 0; - saves layout, does not trigger the event
  • clip: rect(0, 0, 0, 0); - saves layout, does not trigger the event, exotic, should not be used

Preprocessors

There are compilers into CSS to make programmer's life easier. They add inheritance, mixins, variables, loops, etc.

  • SASS - most common
  • Less
  • Stylus

Postprocessors

Manipulate CSS to improve it.

  • Same CSS option may require different prefixes or even different naming. This problem is solved by Autoprefixer
  • Splitted options can be merged into single, saving space. E.g., by shrthnd.
